之前犯了个严重的错误，一直以为双重判断方式的实现是线程安全的，但真实却不是这样的。。。。

先看下双重判断方式的实现代码：

<pre><code>package com.fullstacker.study.designpattern.singleton;

/**
 * 双重判断法
 *
 * @author xingguishuai
 * @create 2017-03-27-16:06
 **/
public class DubbleSingleton {
    private static DubbleSingleton intance ;

    private DubbleSingleton(){

    }
    /**
     * &lt;p&gt;功能描述：判断是否存在存在实例，如果不存在则，创建对象
     * 此时，否则返回实例，为线程不安全&lt;/p&gt;
     * @return
     * @param
     * @author xingguishuai
     * @Date 2017-03-27 15:59
     * @since 1.0
     */
    public static DubbleSingleton getIntance(){
        //第一次检验是否为null
        if(null == intance){
            try {
                //模拟生成对象前准备
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            //同步代码块
            synchronized (DubbleSingleton.class){
                //创建对象前，再次校验是否为null
                if(null == intance){
                    intance = new DubbleSingleton();  <strong>// 此处有可能出现指令重排</strong>
                }
            }
        }
        return intance;
    }

}</code></pre>

java在创建的对象的时候分为以下三个步骤：

<ol>
    <li>分配对象内存（在对象不存在的时候提前占用内存）</li>
    <li>调用构造方法创建对象</li>
    <li>将创建出来的对象指向第一步中分配的内存</li>
</ol>

但是在程序运行期间上面的步骤可能会出现指令重排，造成创建对象的步骤颠倒，如调用对象构造器创建对象和将对象指向分配的内存步骤颠倒，指令执行顺序就会变为：

<ol>
    <li>分配对象内存（在对象不存在的时候提前占用内存）</li>
    <li>将创建出来的对象指向第一步中分配的内存</li>
    <li>调用构造方法创建对象</li>
</ol>

在这种情况下，如果两个线程要获得该单例对象就会出现错误。错误情况如下：

线程A正在创建对象，但发生了上面所说的指令重排，instance先指向了分配的地址。

此时线程B，走到了第一次判断的代码，结果发现instance不为null，所以直接返回了instance，

但由于此时线程A并没有执行对象的构造方法，所以此时并不存在该单例对象。所以程序出现错误

为了解决这个错误就要禁止java程序的指令重排，所以只需要用volatile关键修饰instance变量即可。

修改后的代码，如下：

<pre><code>package com.fullstacker.study.designpattern.singleton;

/**
 * 双重判断法
 *
 * @author xingguishuai
 * @create 2017-03-27-16:06
 **/
public class DubbleSingleton {
    private static <strong>volatile</strong>  DubbleSingleton intance ;  // 利用volatile修饰instance变量，禁止指令重排

    private DubbleSingleton(){

    }
    /**
     * &lt;p&gt;功能描述：判断是否存在存在实例，如果不存在则，创建对象
     * 此时，否则返回实例，为线程不安全&lt;/p&gt;
     * @return
     * @param
     * @author xingguishuai
     * @Date 2017-03-27 15:59
     * @since 1.0
     */
    public static DubbleSingleton getIntance(){
        //第一次检验是否为null
        if(null == intance){
            try {
                //模拟生成对象前准备
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            //同步代码块
            synchronized (DubbleSingleton.class){
                //创建对象前，再次校验是否为null
                if(null == intance){
                    intance = new DubbleSingleton();
                }
            }
        }
        return intance;
    }

}</code></pre>
